The Problem

Every Anchor developer hits these daily

These aren't rare edge cases — they're the primary friction points that slow every Solana engineer every day. No existing AI kit skill covers them.

❌
Cryptic error codes
0x1775, 0x135, custom program error: 309 — what does it mean and where do you look?
⚡
Compute budget exceeded
Transaction fails under load. Where are the CUs going and which change saves the most?
🔄
Unsafe program upgrades
One reordered field corrupts every account. Is your struct change safe to ship?
📍
PDA derivation mismatches
Client and program disagree on the address. Seed encoding, canonical bumps, byte order.
What's Included

Six production-grade skill files

skill/tx-debugging.md
Transaction Debugging
Parse Anchor logs, decode every error code (Anchor, System, Token Program), trace CPI failures through nested instruction calls.
skill/compute-optimization.md
Compute Unit Optimization
Profile CU usage per instruction, identify hogs (large account deserialization, loops, CPIs), implement zero-copy, set accurate budgets.
skill/error-catalog.md
Complete Error Catalog
300+ error codes across Anchor framework, System Program, and Token Program — with causes and fixes, not just names.
skill/upgrade-safety.md
Program Upgrade Safety
Detect breaking struct changes, write migration instructions, IDL diff, upgrade execution and authority management.
skill/simulation.md
Transaction Simulation
Pre-flight checks, accurate CU measurement, batch simulation, CPI failure parsing — all before spending a lamport on mainnet.
skill/common-pitfalls.md
Common Pitfalls
PDA seed encoding, canonical bump storage, account reloading after CPI, overflow, closing accounts safely, duplicate mutables.

Installation

One command install

Via curl
curl -sSL https://raw.githubusercontent.com/nayrbryanGaming/anchor-debugger-skill/master/install.sh | bash
Or clone manually
git clone https://github.com/nayrbryanGaming/anchor-debugger-skill.git && cd anchor-debugger-skill && ./install.sh
As a git submodule
git submodule add https://github.com/nayrbryanGaming/anchor-debugger-skill.git .claude/skills/anchor-debugger-skill
Then use it naturally in Claude Code:
$ /debug-tx <SIGNATURE>
$ /optimize-cu ./programs/my_program/src/lib.rs
$ /check-upgrade ./target/idl/vault.json --program-id <ID>
# or just ask naturally:
"Why did my tx fail with 0x1775?"
