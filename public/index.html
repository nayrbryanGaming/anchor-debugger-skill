The Problem

Two layers of pain, one skill

Payment programs combine all of Anchor's complexity with time-sensitive, money-at-stake constraints that break in uniquely catastrophic ways.

Anchor
❌
Cryptic error codes
0x1775, ConstraintSeeds, AccountDiscriminatorMismatch — decoded to cause + fix instantly.
Anchor
⚡
Compute budget exceeded
Identify exactly which operation is burning CUs and what to change: zero-copy, fixed arrays, CPI batching.
PayFi
🕐
Clock drift
Escrow won't unlock even though wall-clock time has passed. On-chain clock drifts ±5 min from real time.
PayFi
💸
Rent drains stream
Stream account loses rent-exemption and gets garbage collected. All pending payments are lost permanently.
PayFi
🤖
x402 payment rejected
AI agent's payment quote expired before the tx landed. Amount mismatch, wrong mint, replay attack.
PayFi
📦
Batch settlement fails
100 payments queued, 30 fail mid-batch due to CU budget. Priority fees not set. Recipients left unpaid.

Installation

One command

Via curl
curl -sSL https://raw.githubusercontent.com/nayrbryanGaming/anchor-debugger-skill/master/install.sh | bash
Clone + install
git clone https://github.com/nayrbryanGaming/anchor-debugger-skill.git && cd anchor-debugger-skill && ./install.sh
As git submodule
git submodule add https://github.com/nayrbryanGaming/anchor-debugger-skill.git .claude/skills/anchor-payfi-debugger-skill

Then use naturally in Claude Code:

$ /debug-tx <SIGNATURE>
$ /optimize-cu ./programs/escrow/src/lib.rs
$ /check-upgrade ./target/idl/escrow.json --program-id <ID>
# or ask naturally:
"Why did my escrow unlock fail even though the time has passed?"
"Build me a streaming payment with rent protection"
"How do I implement x402 agent payments in Anchor?"
